Ubuntu Linux 安全加固脚本

Bash Ubuntu 20/22

基于CIS Ubuntu Linux 22.04 LTS基准（Level 1 & Level 2）和等保2.0三级要求，自动执行系统加固操作，包含200+条安全配置项。支持 --level 1、--level 2 和 --check-only 模式。

Bash 
#!/bin/bash
# IHCCC Ubuntu 安全加固脚本 v2.1
# 基于 CIS Ubuntu 22.04 LTS Benchmark Level 1

# 检查是否以root运行
[[ $EUID -ne 0 ]] && { echo "需要root权限"; exit 1; }

# 1. 禁用不必要的文件系统类型
echo "install cramfs /bin/true" >> /etc/modprobe.d/cis.conf
echo "install freevxfs /bin/true" >> /etc/modprobe.d/cis.conf
echo "install jffs2 /bin/true" >> /etc/modprobe.d/cis.conf

# 2. 设置 /tmp 挂载选项
systemctl enable tmp.mount

# 3. 禁用IPv6（如不需要）
echo "net.ipv6.conf.all.disable_ipv6 = 1" >> /etc/sysctl.d/60-cis.conf

# 4. 配置SSH安全参数
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^#\?Protocol.*/Protocol 2/' /etc/ssh/sshd_config
sed -i 's/^#\?MaxAuthTries.*/MaxAuthTries 4/' /etc/ssh/sshd_config

# 5. 配置密码策略
sed -i 's/PASS_MAX_DAYS.*/PASS_MAX_DAYS 90/' /etc/login.defs
sed -i 's/PASS_MIN_DAYS.*/PASS_MIN_DAYS 7/' /etc/login.defs
sed -i 's/PASS_WARN_AGE.*/PASS_WARN_AGE 14/' /etc/login.defs

echo "[✓] 基础加固完成，请重启系统生效"

Linux 安全合规审计脚本

Python3 通用Linux

全面扫描Linux系统安全配置状态，生成HTML格式合规报告，涵盖文件权限、网络配置、用户账户、SSH配置、PAM策略等120+检查项，支持CIS等保2.0双标准映射。

Python 
#!/usr/bin/env python3
# IHCCC Linux 安全审计脚本
# 生成CIS/等保2.0合规报告

import subprocess, os, json
from datetime import datetime

CHECKS = {
    "SSH_ROOT_LOGIN": {
        "cmd": "grep '^PermitRootLogin' /etc/ssh/sshd_config",
        "expected": "PermitRootLogin no",
        "cis_id": "5.2.10",
        "level": 1
    },
    "PASSWD_MAX_DAYS": {
        "cmd": "grep '^PASS_MAX_DAYS' /etc/login.defs",
        "expected": "PASS_MAX_DAYS 90",
        "cis_id": "5.4.1.1",
        "level": 1
    }
}

def run_check(name, check):
    result = subprocess.run(check["cmd"], shell=True,
                            capture_output=True, text=True)
    passed = check["expected"] in result.stdout
    return {"name": name, "passed": passed,
            "actual": result.stdout.strip()}

# 运行所有检查并输出报告
results = [run_check(n, c) for n, c in CHECKS.items()]
passed = sum(1 for r in results if r["passed"])
print(f"合规检查完成: {passed}/{len(results)} 项通过")

等保2.0 Linux 自检脚本

Bash 等保三级

针对等保2.0三级要求的Linux系统自检脚本，覆盖身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范等五大技术控制域的100+检查项，输出标准化自检报告。

Bash 
#!/bin/bash
# IHCCC 等保2.0三级 Linux自检脚本
# 覆盖GB/T 22239-2019技术要求

PASS=0; FAIL=0; WARN=0

check() {
  local desc=$1 cmd=$2
  if eval "$cmd" >/dev/null; then
    echo "[✓] $desc"; ((PASS++))
  else
    echo "[✗] $desc"; ((FAIL++))
  fi
}

echo "=== 身份鉴别检查 ==="
check "禁止空密码登录" \
  "grep -q 'PermitEmptyPasswords no' /etc/ssh/sshd_config"
check "启用SSH登录超时" \
  "grep -q 'ClientAliveInterval' /etc/ssh/sshd_config"
check "密码最大有效期≤90天" \
  "awk '/^PASS_MAX_DAYS/{print \$2}' /etc/login.defs | \
   awk '\$1<=90{exit 0}{exit 1}'"

echo "=== 安全审计检查 ==="
check "auditd服务运行中" \
  "systemctl is-active --quiet auditd"
check "记录sudo操作日志" \
  "grep -q 'sudo' /etc/audit/rules.d/*.rules 2>/dev/null"

echo "汇总：通过 $PASS | 失败 $FAIL | 警告 $WARN"

Windows 安全基线检测报告

PowerShell 合规报告

自动检测Windows系统安全配置并生成HTML格式合规报告，涵盖系统配置、账户安全、网络安全、注册表安全设置等80+检查项，支持与CIS基准对照输出通过率。

PowerShell 
# IHCCC Windows 安全检测脚本
# 输出合规检查报告

function Check-SecurityItem {
    param($Name, $ScriptBlock, $Expected, $CisID)
    $actual = & $ScriptBlock
    $pass = $actual -eq $Expected
    [PSCustomObject]@{
        CisID   = $CisID
        Name    = $Name
        Status  = if ($pass) { "通过" } else { "失败" }
        Actual  = $actual
        Expected = $Expected
    }
}

$report = @(
    Check-SecurityItem "最短密码长度" {
        (net accounts | Select-String "Minimum password length") -replace "\D", ""
    } "14" "1.1.4",

    Check-SecurityItem "防火墙状态(域)" {
        (Get-NetFirewallProfile -Name Domain).Enabled
    } "True" "9.1.1"
)

$pass = ($report | Where-Object Status -eq "通过").Count
Write-Host "合规率: $pass/$($report.Count)"
$report | Export-Csv "cis-report-$(Get-Date -f yyyyMMdd).csv" -Encoding UTF8

Docker 安全审计脚本

Bash Docker

基于CIS Docker Benchmark的安全审计脚本，检测Docker主机配置、Docker守护进程安全设置、容器运行时配置、镜像安全、网络配置等方面的合规状态，生成详细审计报告。

Bash 
#!/bin/bash
# IHCCC Docker 安全审计脚本
# 基于 CIS Docker Benchmark

PASS=0; FAIL=0

cis_check() {
  local id=$1 desc=$2 cmd=$3
  if eval "$cmd" >/dev/null 2>&1; then
    echo "[PASS] $id: $desc"; ((PASS++))
  else
    echo "[FAIL] $id: $desc"; ((FAIL++))
  fi
}

echo "=== Docker守护进程配置检查 ==="

# 2.1 限制容器与容器间通信
cis_check "2.1" "禁用容器间通信" \
  "docker info --format '{{.DriverStatus}}' | grep -q icc=false"

# 2.2 设置日志级别
cis_check "2.2" "日志级别为info" \
  "docker info --format '{{.LoggingDriver}}' | grep -qv none"

# 2.5 启用内容信任
cis_check "2.5" "Docker内容信任已启用" \
  "[[ \$DOCKER_CONTENT_TRUST -eq 1 ]]"

# 检查所有运行中容器的配置
for cid in $(docker ps -q); do
  cis_check "5.1" "容器[$cid]非特权运行" \
    "docker inspect $cid --format '{{.HostConfig.Privileged}}' | grep -q false"
done

echo "汇总: 通过 $PASS | 失败 $FAIL"

Kubernetes CIS 合规检测

Bash Kubernetes kube-bench

使用kube-bench工具对Kubernetes集群进行CIS基准合规检测，脚本自动安装kube-bench（支持arm64/amd64），运行全面检测并将报告保存为JSON和文本格式，支持EKS、GKE、AKS等托管集群。

Bash 
#!/bin/bash
# IHCCC Kubernetes CIS 合规检测脚本
# 使用 kube-bench 工具

KUBEBENCH_VERSION="v0.7.3"
REPORT_DIR="/tmp/k8s-cis-report-$(date +%Y%m%d)"

mkdir -p "$REPORT_DIR"

# 安装 kube-bench（如未安装）
if ! command -v kube-bench >/dev/null; then
  ARCH=$(uname -m | sed s/x86_64/amd64/ | sed s/aarch64/arm64/)
  URL="https://github.com/aquasecurity/kube-bench/releases/download"
  curl -L "$URL/$KUBEBENCH_VERSION/kube-bench_${KUBEBENCH_VERSION#v}_linux_$ARCH.tar.gz" \
    | tar -xz -C /usr/local/bin kube-bench
fi

# 运行 CIS Kubernetes 基准检测
echo "[*] 正在检测 Kubernetes 控制平面..."
kube-bench run --targets master \
  --json > "$REPORT_DIR/master-report.json"

echo "[*] 正在检测工作节点..."
kube-bench run --targets node \
  --json > "$REPORT_DIR/node-report.json"

# 提取汇总信息
jq '[.Controls[].tests[].results[] |
  select(.status == "FAIL")] | length' \
  "$REPORT_DIR/master-report.json"
echo "[✓] 报告已保存至: $REPORT_DIR"

Bash + SQL 
#!/bin/bash
# IHCCC MySQL 8.0 安全加固脚本
# 基于 CIS MySQL 8.0 Benchmark

MYSQL_CNF="/etc/mysql/mysql.conf.d/mysqld.cnf"

# 1. 检查MySQL数据目录权限（CIS 1.1）
DATADIR=$(mysql -NBe "SELECT @@datadir" 2>/dev/null)
chmod 750 "$DATADIR" 2>/dev/null && \
  echo "[✓] 数据目录权限设置为750"

# 2. 配置网络绑定（CIS 2.1 - 禁止远程root登录）
mysql -e "DELETE FROM mysql.user WHERE User='root'
  AND Host NOT IN ('localhost','127.0.0.1','::1');"
mysql -e "DELETE FROM mysql.user WHERE User='';"
mysql -e "FLUSH PRIVILEGES;"
echo "[✓] 已清理匿名账户和远程root"

# 3. 配置密码策略（CIS 2.7）
mysql -e "SET GLOBAL validate_password.policy=MEDIUM;"
mysql -e "SET GLOBAL validate_password.length=14;"

# 4. 启用审计日志（CIS 2.8）
echo "general_log = 0" >> $MYSQL_CNF
echo "log_error = /var/log/mysql/error.log" >> $MYSQL_CNF
echo "slow_query_log = 1" >> $MYSQL_CNF

echo "[✓] MySQL安全加固完成，请重启MySQL服务"

云平台安全检测脚本（AWS/阿里云）

Python3 云安全

基于CIS AWS/阿里云基准的云平台安全检测脚本，通过云SDK检测IAM配置（MFA启用、root账户使用）、S3/OSS存储桶权限、CloudTrail日志记录、安全组规则等关键安全配置。

Python (AWS) 
#!/usr/bin/env python3
# IHCCC AWS 安全基线检测脚本
# 基于 CIS Amazon Web Services Benchmark

import boto3, json

iam = boto3.client('iam')
s3  = boto3.client('s3')

def check_mfa_root():
    """CIS 1.5: 确保root账户已启用MFA"""
    summ = iam.get_account_summary()['SummaryMap']
    mfa_on = summ.get('AccountMFAEnabled', 0) == 1
    return {"cis": "1.5", "name": "Root账户MFA", "pass": mfa_on}

def check_s3_public_access():
    """CIS 2.1.5: 确保S3存储桶阻止公共访问"""
    issues = []
    for b in s3.list_buckets()['Buckets']:
        try:
            cfg = s3.get_public_access_block(Bucket=b['Name'])
            blk = cfg['PublicAccessBlockConfiguration']
            if not all(blk.values()):
                issues.append(b['Name'])
        except: issues.append(b['Name'])
    return {"cis": "2.1.5", "name": "S3公共访问阻止",
            "pass": len(issues)==0, "issues": issues}

results = [check_mfa_root(), check_s3_public_access()]
for r in results:
    status = "✓ 通过" if r["pass"] else "✗ 失败"
    print(f"[{status}] CIS {r['cis']}: {r['name']}")

Ansible 批量加固 Playbook

Ansible Linux批量

使用Ansible Playbook实现Linux服务器批量安全加固，基于CIS基准的Role设计，支持同时对数十至数百台服务器执行统一安全配置，适合企业规模化安全部署场景。

Ansible YAML 
# IHCCC Ansible CIS 加固 Playbook
# 文件: cis-hardening.yml
---
- name: IHCCC CIS Linux 安全加固
  hosts: all
  become: yes
  vars:
    cis_level: 1
    cis_ssh_port: 22
    cis_pass_max_days: 90
    cis_pass_min_len: 14

  tasks:
    - name: CIS 5.2.10 - 禁止root SSH登录
      lineinfile:
        path: /etc/ssh/sshd_config
        regexp: '^PermitRootLogin'
        line: 'PermitRootLogin no'
        state: present
      notify: restart sshd

    - name: CIS 5.4.1.1 - 设置密码最大有效期
      lineinfile:
        path: /etc/login.defs
        regexp: '^PASS_MAX_DAYS'
        line: "PASS_MAX_DAYS {{ cis_pass_max_days }}"

    - name: CIS 4.1.2 - 确保auditd服务运行
      service:
        name: auditd
        state: started
        enabled: yes

  handlers:
    - name: restart sshd
      service:
        name: sshd
        state: restarted
