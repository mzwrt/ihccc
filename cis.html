什么是 CIS 安全基准？

CIS Benchmarks（互联网安全中心安全基准）是由全球安全专家社区共同制定的免费安全配置指南，适用于操作系统、中间件、应用程序和云基础设施。

🌍

全球公认

CIS基准被全球数千家企业、政府机构和安全团队采用，是信息安全领域最权威的配置加固标准之一，被NIST、PCI-DSS、HIPAA等主流框架引用。

🆓

完全免费

所有CIS基准文档可免费下载（PDF格式），覆盖Windows、Linux、macOS、云平台、容器、数据库等主流技术栈，无需付费订阅即可获取全部内容。

🔄

持续更新

CIS基准随技术版本更新而迭代，由全球安全专家通过开放共识流程审核维护，确保配置建议与最新安全威胁和技术发展保持同步。

📊

分级实施

每个基准分为 Level 1（基础安全，最小影响）和 Level 2（高安全，适合高敏感环境）两个配置级别，企业可根据风险需求灵活选择实施深度。

⚙️

可自动化

CIS基准支持通过CIS-CAT工具、Ansible Playbook、Chef、Puppet、Terraform等自动化工具进行合规检查与批量部署，大幅降低实施成本。

🇨🇳

适配中国合规

CIS基准与中国等保2.0标准高度兼容，实施CIS基准可同时满足等保2.0的大部分技术要求，是国内企业进行安全加固的高效路径。

CIS Controls v8 — 18项核心安全控制

CIS Controls是针对网络防御的优先级安全行动，分为三个实施组（IG1基础、IG2中级、IG3高级），帮助各类规模的组织构建有效的安全防御体系。

💡 实施组说明：IG1（基础保护，适合小型企业）→ IG2（中级防护，适合中型组织）→ IG3（高级防护，适合高风险/关基单位）。每个控制包含若干具体的安全活动（Safeguards）。

CIS Control 1IG1

企业资产清单与控制

主动管理（清点、跟踪、纠正）所有企业资产，以确保只有授权设备才能访问企业网络资源。

CIS Control 2IG1

软件资产清单与控制

主动管理（清点、跟踪、纠正）企业中所有软件，确保只有授权软件被安装和运行。

CIS Control 3IG1

数据保护

制定流程和技术控制来识别、分类、安全处理、保留和处置数据，防止数据未经授权的访问、修改或泄露。

CIS Control 4IG1

企业资产与软件的安全配置

建立和维护企业资产（终端、服务器、网络设备）和软件的安全配置，防止攻击者利用易受攻击的服务和设置。

CIS Control 5IG1

账户管理

使用流程和工具来分配和管理企业资产和软件上的账户授权，包括管理员账户。

CIS Control 6IG1

访问控制管理

使用流程和工具来创建、分配、管理和撤销企业资产和软件的访问凭证和权限，实施最小权限原则。

CIS Control 7IG1

持续漏洞管理

制定计划，持续评估和跟踪企业资产中已知漏洞的修复情况，并通过风险评估优先处理漏洞修复。

CIS Control 8IG1

审计日志管理

收集、分析和保留企业资产的审计日志，以检测、理解和响应攻击事件，确保日志完整性和可用性。

CIS Control 9IG1

电子邮件和Web浏览器保护

通过提升用户意识并使用技术手段，改善攻击者利用Web浏览器和电子邮件客户端来攻击用户的防护能力。

CIS Control 10IG1

恶意软件防御

防止在企业资产上安装、传播和运行恶意代码，同时优化自动化工具的使用以快速更新防御、收集数据和纠正行动。

CIS Control 11IG2

数据恢复

建立和维护对企业资产进行完整数据恢复的实践，确保在安全事件后能够及时恢复业务运营。

CIS Control 12IG2

网络基础设施管理

建立、实施和主动管理（跟踪、报告、纠正）网络设备，以防止攻击者利用易受攻击的网络服务和访问控制点。

CIS Control 13IG2

网络监控与防御

操作流程和工具以建立全面的网络监控与防御能力，抵御以网络为核心的攻击活动。

CIS Control 14IG2

安全意识和技能培训

建立并维护安全意识教育计划，以影响员工在日常工作中的安全行为，降低人为因素引发的安全风险。

CIS Control 15IG2

服务提供商管理

制定流程来评估服务提供商，管理其与企业的关系，确保服务提供商能够满足企业适当的安全要求。

CIS Control 16IG2

应用软件安全

管理企业开发、采购或外包的应用程序的安全生命周期，以检测和纠正安全弱点，防止攻击者利用它们。

CIS Control 17IG3

事件响应管理

建立一个程序来准备、检测、遏制、分析、清除、恢复和协调对网络事件的响应，并通知相关方。

CIS Control 18IG3

渗透测试

通过模拟攻击者的目标和行动来测试企业防御的强度，评估现有安全控制的有效性，发现薄弱环节。

基准配置级别说明

每个CIS基准通常包含两个配置级别，以及可选的 Workstation 和 Server 两种配置文件。

配置级别 适用场景 安全性 业务影响 推荐对象
Level 1 所有环境（通用基础） 基础安全 最小影响（对业务基本无干扰） 所有企业，可作为最低安全基线
Level 2 高安全需求环境 高强度安全 部分功能受限（需测试后部署） 金融、医疗、政府、关基等高敏感行业
STIG 美国政府/军事环境 最高安全 较大影响（需专业实施） 国防、情报、联邦政府机构（DoD要求）
NextGen 新兴技术环境 前沿安全 因场景而异 云原生、DevSecOps、零信任架构
